# План: Обработка составных регионов и пост-анализ датасета

## Суть задачи

Два уровня логики:

1. **Уровень строки** — если в одной строке два региона через "и", обычно возвращаем `(None, None)`. Исключение: Архангельская+НАО и Тюменская+ХМАО/ЯНАО.
2. **Уровень датасета** — после нормализации всех строк: анализируем множество найденных регионов и уточняем, имеется в виду область С автономным округом или БЕЗ.

## Этап 2: Обработка составных строк (compound input)

### Новый метод `_handle_compound_input` в `RegionMatcher`

**Алгоритм:**
1. Препроцессить строку через `preprocess_name`
2. Проверить, не является ли полная строка аббревиатурой (если да — вернуть `None`, пусть обычный матчинг работает)
3. Если в строке есть ` и ` — разбить по ` и ` и `,`
4. Каждую часть нормализовать через fuzzy matching (через `_find_best_match_core` — без вызова compound check, чтобы избежать рекурсии)
5. Сверить набор распознанных регионов с правилами исключений:
   - {Архангельская*, Ненецкий АО} → "Архангельская область (с Ненецким АО)"
   - {Тюменская*, ХМАО} → "Тюменская область (с автономными округами)"
   - {Тюменская*, ЯНАО} → "Тюменская область (с автономными округами)"
   - {Тюменская*, ХМАО, ЯНАО} → "Тюменская область (с автономными округами)"
6. Если правило не сработало — вернуть `(None, None)`

### Новая константа `COMPOUND_REGION_RULES` в `constants.py`

```python
COMPOUND_REGION_RULES = {
    frozenset({"Архангельская область (с Ненецким АО)", "Ненецкий автономный округ"}):
        "Архангельская область (с Ненецким АО)",
    frozenset({"Архангельская область без автономного округа", "Ненецкий автономный округ"}):
        "Архангельская область (с Ненецким АО)",
    frozenset({"Тюменская область (с автономными округами)", "Ханты-Мансийский автономный округ — Югра"}):
        "Тюменская область (с автономными округами)",
    frozenset({"Тюменская область (с автономными округами)", "Ямало-Ненецкий автономный округ"}):
        "Тюменская область (с автономными округами)",
    frozenset({"Тюменская область (с автономными округами)", "Ханты-Мансийский автономный округ — Югра", "Ямало-Ненецкий автономный округ"}):
        "Тюменская область (с автономными округами)",
    frozenset({"Тюменская область без автономных округов", "Ханты-Мансийский автономный округ — Югра"}):
        "Тюменская область (с автономными округами)",
    frozenset({"Тюменская область без автономных округов", "Ямало-Ненецкий автономный округ"}):
        "Тюменская область (с автономными округами)",
    frozenset({"Тюменская область без автономных округов", "Ханты-Мансийский автономный округ — Югра", "Ямало-Ненецкий автономный округ"}):
        "Тюменская область (с автономными округами)",
}
```

### Рефакторинг `find_best_match`

Вынести тело fuzzy matching в `_find_best_match_core` (тот же код, но без compound check). `find_best_match` становится:

```python
def find_best_match(self, input_name, ...):
    compound_result = self._handle_compound_input(input_name)
    if compound_result is not None:
        return compound_result
    return self._find_best_match_core(input_name, ...)
```

## Этап 3: Пост-анализ датасета (resolve ambiguity)

### Новый метод `_resolve_parent_regions` в `RegionMatcher`

**Логика (вызывается после нормализации всех строк в `match_dataframe`):**

После того как все уникальные значения нормализованы, проверяем:

1. **Архангельская область:**
   - Если в результатах есть "Ненецкий автономный округ" И "Архангельская область (с Ненецким АО)" → заменить на **"Архангельская область без автономного округа"** + WARNING
   - Если есть "Архангельская область (с Ненецким АО)" БЕЗ "Ненецкий АО" → оставить как есть + INFO-сообщение
   - Если "Архангельская область без автономного округа" — оставить как есть

2. **Тюменская область:**
   - Если в результатах есть любой из {ХМАО, ЯНАО} И "Тюменская область (с автономными округами)" → заменить на **"Тюменская область без автономных округов"** + WARNING
   - Если "Тюменская область (с автономными округами)" БЕЗ ХМАО/ЯНАО → оставить + INFO
   - Если "Тюменская область без автономных округов" — оставить как есть

### Формат предупреждений

```
INFO: "Архангельская область" распознана как "Архангельская область (с Ненецким АО)" —
  Ненецкий АО отдельно в данных не обнаружен.
WARNING: "Архангельская область" переопределена на "Архангельская область без автономного округа" —
  Ненецкий АО обнаружен отдельно в данных.
```

### Где вызывается

В методе `match_dataframe` — **после** цикла нормализации уникальных значений, но **перед** маппингом на DataFrame:

```python
def match_dataframe(self, df, column_name, **kwargs):
    # ... существующий код нормализации уникальных значений ...

    # NEW: пост-анализ для Архангельской/Тюменской
    value_mapping = self._resolve_parent_regions(value_mapping)

    # ... существующий код маппинга на DataFrame ...
```

### Метод `_resolve_parent_regions`

Принимает `value_mapping` (dict: исходное_имя → нормализованное_имя), анализирует множество нормализованных имён, при необходимости заменяет и выводит WARNING/INFO.

## Этап 5: Тесты

### Тесты для compound input:
- "город Москва и Московская область" → `(None, None)`
- "Санкт-Петербург и Ленинградская область" → `(None, None)`
- "Архангельская область и Ненецкий АО" → `("Архангельская область (с Ненецким АО)", 100.0)`
- "Тюменская область и ХМАО" → `("Тюменская область (с автономными округами)", 100.0)`

### Тесты для пост-анализа (match_dataframe):
- DataFrame с ["Архангельская область", "Ненецкий АО", "Москва"] → Архангельская → "Архангельская область без автономного округа"

- DataFrame с ["Тюменская область", "ХМАО", "ЯНАО"] → Тюменская → "Тюменская область без автономных округов"
- DataFrame с ["Тюменская область", "Москва"] (без ХМАО/ЯНАО) → "Тюменская область (с автономными округами)"

### Тесты для find_best_match (одиночные строки):
- "Архангельская область" → "Архангельская область (с Ненецким АО)" (по умолчанию — с АО, т.к. контекста датасета нет)
- "Тюменская область" → "Тюменская область (с автономными округами)"
